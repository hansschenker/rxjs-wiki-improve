## Observable (Quick Version) [(00:02:39)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=159s)
- Observables are sets of values over time, and when subscribed to, they take an observer or a set of functions, wrap it in an observer object, and tie it to a data-producing source, such as a for loop, websocket, or anything else, allowing for the use of expensive resources like websockets that can be closed when no longer needed [(00:02:39)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=159s).
- On completion or when unsubscribing, the observable is able to tear down the observer, enabling the abrupt stopping of animations or other expensive operations, and the most important aspect of observables is that they are sets, which makes them powerful [(00:03:08)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=188s).
- Sets have certain properties, such as being able to concatenate and having an identity property, and operators are functions that transform one observable set into another, allowing for mapping, filtering, flattening, and joining, which are essential in reactive programming [(00:03:44)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=224s).
- Thinking reactively involves transforming sets of events, starting with one set and trying to transform it into the desired set of events, rather than just reacting to individual events, and this concept is crucial when working with RxJS [(00:04:52)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=292s).
- Animations are a significant source of sets of events in an application, and while they are sometimes viewed as non-essential, they are technically interesting and important, with talks by experts like Sarah Drasner highlighting their importance, and this topic will be explored in more depth [(00:05:31)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=331s).
- RxJS provides methods for working with animations, and understanding how to use observables and operators to transform sets of events is essential for creating complex animations and managing state in applications [(00:06:02)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=362s).

## Animation (defined) [(00:07:27)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=447s)
- Animation is defined as the technique of photographing successive drawings or positions of puppets or models to create an illusion of movement when the movie is shown as a sequence, and in web development, this concept can be applied to sets of positions over time [(00:07:27)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=447s).
- The concept of animation can be represented as an observable, where an object's movement can be broken down into snapshots of its position at different points in time, such as moving from position zero to position one, and this can be applied to various properties like rotation and scale [(00:08:11)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=491s).
- Observables can be used to represent values over time, but they lack the temporal element of time, which is essential for animations, and this is where the concept of time becomes crucial in animations [(00:08:49)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=529s).
- There are two elements of time involved in animations: frame rate and duration or velocity, where frame rate refers to the frequency at which the position of an object is adjusted before re-rendering, and duration or velocity refers to the speed at which the object moves [(00:09:14)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=554s).
- A frame is a moment in time at which the position of an object is adjusted before re-rendering, and multiple frames make up a movie, similar to a film strip, which consists of multiple frames [(00:09:42)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=582s).

## requestAnimation Frame [(00:09:58)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=598s)
- Request animation frame is an API used in modern web browsers to handle animations, where a callback is provided to be fired when the browser is ready to render, allowing for updates and scheduling of animations to be rendered [(00:09:58)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=598s).
- When calling request animation frame, it informs the browser that some animation code should be run when the browser is not busy and is about to start a render cycle, enabling the scheduling of updates and animations [(00:10:10)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=610s).
- The request animation frame API can return an ID that can be used to cancel the animation later if needed, and it is similar to set timeout, with RxJS having a wrapper called animation frame scheduler that handles this [(00:10:29)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=629s).
- The animation frame scheduler in RxJS has a schedule method that takes a function to be fired on the animation frame and returns an RxJS subscription that can be used to cancel the animation frame [(00:10:42)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=642s).
- Schedulers in RxJS, such as the animation frame scheduler, provide a way to schedule functions to be executed at specific times or intervals, with the animation frame scheduler being a simple API that allows for the scheduling of animations [(00:10:49)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=649s).
- The schedule method of the animation frame scheduler returns an RxJS subscription that can be used to cancel the animation frame, although it is rare to use schedulers directly in RxJS [(00:11:00)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=660s).

## What is a Scheduler in Rx? [(00:11:16)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=676s)
- Schedulers in RxJS control the timing of when observable events fire, acting as a gating mechanism that determines when values are allowed to come through to the handler, and they can be used to wait for specific events, such as an animation frame, before providing a value [(00:11:16)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=676s).
- When subscribing to an observable, schedulers can delay the execution of the subscription and the firing of events until the scheduler allows it, providing a way to control the flow of values and errors [(00:11:24)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=684s).
- Schedulers are available as the last argument in most observable creation methods and can be used to specify when an observable should subscribe and fire events, allowing for more control over the timing of observable events [(00:12:09)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=729s).
- There are also operators available in RxJS that allow for scheduling, such as observing on a specific schedule or subscribing on a particular schedule, providing more flexibility in using schedulers with observables [(00:12:55)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=775s).
- Schedulers are not part of the TC39 Proposal, but are a key feature in RxJS, and there are several different schedulers available to try out, each with its own use cases and applications [(00:12:23)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=743s).

## Endless stream of frames for animations [(00:13:46)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=826s)

- The `observable.range` method can create an instantaneous observable range, but using it with an infinite upper bound without scheduling can lock the browser, so instead, `animationFrame` is used to pause between each frame, preventing the browser from locking up, [(00:14:43)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=883s).

- To handle this issue, instead of animating by frame number, a better approach is needed, and having an observable of frames is the first step, allowing for the exploration of other aspects of the time equation, such as velocity or duration, which are related through a simple algorithm, [(00:16:44)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1004s).

- The key characteristic of animating by velocity is that it does not have a predefined end time, allowing the animation to potentially last forever, which is particularly useful for applications that require continuous motion or animation [(00:17:44)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1064s).

## Animate by duration [(00:18:02)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1082s)

## Recap: Velocity-based Animations [(00:22:23)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1343s)
- Velocity-based animations are simple and involve mapping time differences into position differences based on velocity using basic math, making them the simplest way to work with animations, allowing for movement by a certain amount over a specific time period [(00:22:23)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1343s).
- Duration-based animations can be created by taking the velocity-based animation and adding a take while condition, which allows the animation to move at a certain velocity for a specified duration, such as two seconds, although this may not be the best solution [(00:22:54)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1374s).
- A better approach to duration-based animations is to treat durations as percentages, or decimal values between zero and one, which provides flexibility and allows for easy transformation and scaling of the animation, regardless of the duration [(00:23:40)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1420s).
- By using percentages, the range of values for the animation is always between zero and one, making it easier to work with the observable and providing a consistent range of values, whether the duration is 10 seconds or 5 milliseconds [(00:24:02)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1442s).
- Building a duration observable based on this approach can provide a more effective way to manage animations, allowing for greater control and flexibility over the animation [(00:24:27)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1467s).

## Build a duration observable [(00:24:32)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1472s)
- The milliseconds elapsed function, previously created for velocity, is utilized to build a duration observable by mapping it to a duration function that takes a milliseconds variable, representing the desired animation duration, and then dividing the elapsed time by the desired duration milliseconds, with the result being taken while it is less than or equal to one [(00:24:32)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1472s).
- The duration function allows for the scheduler to be passed through, enabling potential testing later on, and the outcome is a range of values starting at zero, progressing through arbitrary decimal amounts as animation frames fire, and ultimately reaching one at the end [(00:24:42)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1482s).
- The resulting observable emits a range of values, from zero to one, over the specified duration, with the values being emitted non-deterministically as animation frames fire, allowing for the creation of animations that move over a distance [(00:25:18)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1518s).

## Moving over a distance is simple multiplication [(00:25:36)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1536s)
- Moving over a distance becomes very simple multiplication when the duration is defined as the time it takes to reach a value of one, and the distance to be traveled is multiplied by this value to determine the current position at any given time [(00:25:36)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1536s).
- The calculation involves multiplying the percentage of the duration that has passed by the total distance to be traveled, so if the duration is halfway complete, the current position would be at 50% of the total distance [(00:25:51)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1551s).
- This simple math allows for precise calculation of the current position, for example, if the distance to be traveled is 300 pixels and the duration is halfway complete, the current position would be at 150 pixels, which is exactly where it should be [(00:26:01)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1561s).

- Easing refers to the process of modifying an animation to make it more engaging, such as making an object bounce into place, slowly come to a stop, or speed up and accelerate, and this can be achieved by applying different easing functions to the animation [(00:27:34)](https://www.youtube.com/watch?v=X_RnO7KSR-4&t=1654s).

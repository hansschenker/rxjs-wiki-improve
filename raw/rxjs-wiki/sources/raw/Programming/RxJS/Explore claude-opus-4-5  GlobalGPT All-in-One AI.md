## Introduction to RxJS and Core Concepts
- RxJS, which stands for [[ReactiveX | Reactive Extensions]] for [[JavaScript]], is a library for reactive programming using Observables, making it easier to compose asynchronous or callback-based code by providing a paradigm centered around data streams and the propagation of change.
- The core concept of reactive programming in RxJS involves key building blocks, including Observables, which are streams of data or events over time, Observers, which consume or listen to the data emitted, Subscriptions, which connect Observables and Observers, Operators, which are pure functions used to transform streams, Subjects, which are both Observables and Observers, and Schedulers, which control concurrency and timing.
- A basic example of using RxJS involves creating an Observable, subscribing to it with operators such as filter and map, and handling the emitted values, errors, and completion, demonstrating how RxJS can simplify complex asynchronous code.
- RxJS provides various operators, including Creation Operators like of, from, interval, and fromEvent, Transformation Operators like map, switchMap, mergeMap, and concatMap, and Filtering Operators like filter, take, debounceTime, and distinctUntilChanged, which can be used to manipulate and process data streams.

## Core Building Blocks of Reactive Programming
- A real-world use case of RxJS is handling search input, where it can be used to debounce user input, avoid duplicate searches, and fetch new results, showcasing its ability to handle complex async flows and real-time data.
- RxJS has several advantages over [[Futures and promises | Promises]], including the ability to emit multiple values, lazy execution, cancellability, and a wide range of operators, making it a more powerful tool for handling asynchronous code.
- RxJS is widely used in various frameworks and libraries, including Angular, React, Node.js, and state management tools like NgRx and Redux-Observable, demonstrating its versatility and effectiveness in handling complex async flows and real-time data.

## Design Patterns and Origins of RxJS
- The roots of RxJS are deeply rooted in decades of computer science, mathematics, and software design, with key influences including the [[Observer pattern | Observer Pattern]], [[Iterator pattern | Iterator Pattern]], Functional Programming, Functional Reactive Programming, and [[ReactiveX | Reactive Extensions]].
- The Observer Pattern, introduced in the Gang of Four Design Patterns book in 1994, is a fundamental concept in RxJS, where an object maintains a list of dependents and notifies them of state changes, similar to how an Observable in RxJS works with lifecycle management.
- The Iterator Pattern, also from the Gang of Four book, provides a way to sequentially access elements without exposing the underlying structure, and its connection to RxJS is that an Observable is the "push" dual of an Iterator, where the producer pushes values instead of the consumer pulling them.
- Functional Programming, which originated in the 1950s, has directly shaped RxJS, with core principles such as pure functions, composition, higher-order functions, immutability, and declarative style being manifested in RxJS operators, pipes, and streams.
- Functional Reactive Programming, introduced by Conal Elliott and Paul Hudak in 1997, is another key root of RxJS, with its core abstractions of Behaviors and Events, although RxJS is closer to "reactive programming" than true FRP due to its handling of discrete event streams.
- The direct parent of RxJS is Reactive Extensions, developed by [[Erik Meijer (computer scientist) | Erik Meijer]] at Microsoft in 2009, which has led to the creation of various implementations, including RxJS, Rx.NET, RxJava, RxPy, and RxSwift, with RxJS evolving from version 4 to the modern, tree-shakable versions 5, 6, and 7.

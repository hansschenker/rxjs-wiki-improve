## RxJS Architecture and Observables
- The RxJS library allows for the same operators to be used across different domains, such as Web animations and [[Global Positioning System | GPS]] sensor data, due to its underlying architecture, which treats everything as a stream of values over time, represented by the Observable interface.
- The Observable interface is a universal abstraction that merges the Observer pattern with the Iterator pattern, creating a "push-based collection" where values arrive over time, and it is defined by the contract that includes the subscribe method, which takes an observer as an argument and returns a subscription.
- RxJS operators are pure functions that transform Observables into Observables, essentially acting as endofunctors, which means they work on the structure of the stream, not the semantics of the data, allowing them to be used across different domains without modification.
- The key insight behind RxJS is that the domain-specific part is only in the Observable creation, which is the producer, while everything after that, including the operators, is pure stream algebra, making it possible to reuse operators across different domains.

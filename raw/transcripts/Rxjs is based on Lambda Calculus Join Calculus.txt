    const latest = new Array(n);
    const hasValue = new Array(n).fill(false);
    let ready = 0;
    let active = n;

    const subscriptions = sources.map((source, i) =>
      source.subscribe({
        next(value) {
          latest[i] = value;

          if (!hasValue[i]) {
            hasValue[i] = true;
            ready++;

          if (ready === n) {
            subscriber.next([...latest]);
          }

          subscriber.error(err);
        },
        complete() {
          active--;
          if (active === 0) {
            subscriber.complete();
          }
        }
      })
    );

    return () => {
      subscriptions.forEach(s => s.unsubscribe());
    };
  });
}

RxJS, FRP, Join Calculus — A Unified Reference
1. Big Picture
RxJS is best understood as a practical synthesis of:

Lambda Calculus → pure transformations
Monads → sequencing of effectful computations
Join Calculus → coordination of concurrent events
FRP (Functional Reactive Programming) → lifting coordination rules into time
Key insight:
RxJS is not “about streams”; it is about maintaining invariants over time.

2. Observables as Effectful, Temporal Values
Observable Mental Model
Copy
Observable<T> ≈ Effect<T> over Time
Effects are encapsulated (lazy, deferred)
Nothing happens until subscription
Effects are composed, not executed directly
This eliminates callback hell by:

Removing inversion of control
Making time explicit
Unifying error, completion, and cancellation
3. Monadic Layer (Sequential Effects)
Observables behave monad‑like:

Monad Concept	RxJS
Type constructor	Observable<T>
pure	of(value)

What monads give RxJS:

Linear sequencing of async effects
Referential transparency at the composition level
Limitation:

Monads alone cannot express concurrent coordination
4. Join Calculus Layer (Concurrency & Coordination)
Join calculus models coordination with rules like:

Copy
when A(x) & B(y) => P(x, y)
RxJS join operators encode these rules explicitly:

combineLatest
zip
withLatestFrom
forkJoin
They define when computation is allowed to occur across multiple sources.

5. FRP: Lifting Join Calculus into Time
The Lift
Join calculus (atemporal):

Copy
when A & B => emit
FRP (time‑lifted):

Copy
∀ t, when A(t) & B(t) => emit(A(t), B(t))
Key transformation:

Messages become persistent, time‑indexed values
Join rules become continuously maintained invariants
This is why RxJS operators re‑emit as values change.

Formal Invariant

Never emits partial tuples

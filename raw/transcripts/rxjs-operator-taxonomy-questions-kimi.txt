Follow-up questions to narrow down:

Question	If Yes	If No
"Is the reduction based on position/index?"	take(n), skip(n), takeLast(n), skipLast(n)	Continue →
"Is the reduction based on a condition/predicate?"	filter (keep matching), takeWhile (stop when false), skipWhile (start when false)	Continue →
"Do you only need a specific single element?"	first (first match or default), last (requires completion), single (exactly one or error), elementAt(index)	Continue →
"Is the reduction based on equality/uniqueness?"	distinct (all unique, unbounded memory), distinctUntilChanged (consecutive unique, O(1) memory), distinctUntilKeyChanged (by key)	Continue →
"Do you want to drop ALL values and only keep completion/error?"	ignoreElements (N→0)	Continue to other branches

RxJS Operators (Multi-Dimensional Taxonomy)
│
├── Value-Based ───────────────────────────────────────────────────────────┐
│   │                                                                       │
│   ├── Lossy ──────────────────────────────────────────────────────────┐  │
│   │   │                                                                │  │

│   │   │   │                                                         │  │  │
│   │   │   ├── Filtering ─────────────────────────────────────────┐  │  │  │
│   │   │   │   Q: "Do you need FEWER emissions based on position, 
│   │   │   │      predicate, or specific element selection?"
│   │   │   │   
│   │   │   │   └─→ Position-based? 
│   │   │   │       → "From start or end?" → take/skip | takeLast/skipLast
│   │   │   │       → "Until condition signals?" → takeUntil/skipUntil
│   │   │   │       → "While condition holds?" → takeWhile/skipWhile
│   │   │   │
│   │   │   │   └─→ Predicate-based?
│   │   │   │       → "Keep matches?" → filter
│   │   │   │       → "Extract exactly one?" → first/last/single/elementAt
│   │   │   │
│   │   │   └── Distinct ────────────────────────────────────────┘  │  │  │
│   │   │       Q: "Do you need to REMOVE DUPLICATE values?"
│   │   │       
│   │   │       └─→ "All-time uniqueness (growing memory)?" → distinct
│   │   │       └─→ "Only consecutive duplicates?" → distinctUntilChanged
│   │   │       └─→ "By object key?" → distinctUntilKeyChanged
│   │   │
│   │   ├── Cardinality: N→1 (Aggregate/Fold) ─────────────────────┘  │  │
│   │       Q: "Do you need to COMPRESS all values into ONE result?"
│   │       
│   │       └─→ "Wait for completion, emit final?" → reduce
│   │       └─→ "Count items?" → count
│   │       └─→ "Find min/max?" → min/max
│   │       └─→ "Check ALL satisfy condition?" → every
│   │       └─→ "Find first match index/value?" → find/findIndex
│   │       └─→ "Is stream empty?" → isEmpty
│   │
│   │   └── Higher-Order: Canceling/Switching ────────────────────────┘  │
│           Q: "Do you map to inner observables that should CANCEL

│           
│           └─→ "Cancel previous when new arrives?" → switchMap/switchAll
│           └─→ "Ignore new until current completes?" → exhaustMap/exhaustAll
│           └─→ "Scan with switch behavior?" → switchScan
│           └─→ "Race multiple, first wins?" → race
│
│   └── Non-Lossy ──────────────────────────────────────────────────────┐ │
│       │                                                                │ │
│       ├── Cardinality: 1→1 (Transform) ────────────────────────────┐  │ │
│       │   Q: "Do you need to TRANSFORM each value to something else?"
│       │   
│       │   └─→ "Apply function?" → map
│       │   └─→ "Extract property?" → pluck
│       │   └─→ "Map to constant?" → mapTo

│       │

│           └─→ "Emit intermediate states?" → scan

│       ├── Cardinality: 1→∞ (Unfold) ───────────────────────────────┤  │ │

│       └── Buffer/Window Static ────────────────────────────────────┘  │ │

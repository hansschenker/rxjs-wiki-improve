Input / Output value contract
does one input value produce zero, one, or many output values?
does the operator preserve type shape or change it?
does it lift values into inner streams?
does it combine multiple sources?
does it remember prior values?

This is important because:
map = 1 input -> 1 output
filter = 1 input -> 0 or 1 output
scan = 1 input + memory -> 1 accumulated output
switchMap = 1 outer input -> inner stream -> many possible outputs
bufferTime = many inputs over time -> array output

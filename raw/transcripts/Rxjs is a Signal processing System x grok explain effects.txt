What are Effects in NgRx?NgRx is a state management library for Angular inspired by Redux, and it heavily leverages RxJS for reactive programming. In NgRx, the core principle is that reducers must be pure functions: they take the current state and an action, and return a new state without performing side effects (like API calls, logging, navigation, or timers).Effects (from @ngrx/effects) are the dedicated place to handle side effects in a predictable, testable, and centralized way. They allow you to listen to dispatched actions, perform asynchronous or impure operations, and then dispatch new actions based on the results.Key Role of EffectsIsolate side effects: Keep reducers pure and components clean.
Handle asynchronous tasks: Most commonly used for API calls (e.g., fetching data), but also for WebSockets, local storage, analytics, navigation, etc.
Reactive with RxJS: Effects are built on Observables, so you can use powerful RxJS operators (e.g., switchMap, mergeMap, catchError, debounceTime).

How Effects WorkAn action is dispatched (e.g., from a component: store.dispatch(loadUsers())).
Reducers run first: They synchronously update the state based on the action (NgRx guarantees this order).
Effects then listen to the stream of all dispatched actions (via an injected Actions Observable).
If the action matches (using ofType), the effect executes its logic.
The effect typically returns a new Observable of actions (e.g., success or failure), which NgRx automatically dispatches back to the store.

Effects run after reducers, so they can access the updated state if needed (via selectors or operators like withLatestFrom or NgRx's concatLatestFrom).Basic ExampleHere's a typical effect for loading users from an API:typescript

import { Injectable } from '@angular/core';
import { Actions, createEffect, ofType } from '@ngrx/effects';
import { of } from 'rxjs';
import { map, mergeMap, catchError } from 'rxjs/operators';
import { UserService } from './user.service';
import * as UserActions from './user.actions';

@Injectable()
export class UserEffects {

  loadUsers$ = createEffect(() => this.actions$.pipe(
    ofType(UserActions.loadUsers),  // Listen for this action
    mergeMap(() => this.userService.getUsers().pipe(
      map(users => UserActions.loadUsersSuccess({ users })),  // Dispatch success
      catchError(error => of(UserActions.loadUsersFailure({ error })))  // Dispatch failure
    ))
  ));

  constructor(
    private actions$: Actions,
    private userService: UserService
  ) {}
}

createEffect() marks this as an effect (modern NgRx uses this functional style; older versions used @Effect() decorator).
By default, effects dispatch the returned actions ({ dispatch: true }).
You can create non-dispatching effects (e.g., for navigation or logging) with { dispatch: false }.

What are Effects in NgRx?NgRx is a reactive state management library for Angular, built on principles from Redux and heavily utilizing RxJS for handling asynchronous data flows. Effects are a key part of the @ngrx
/effects package, designed to manage side effects—operations that interact with the outside world, such as API calls, browser storage, or routing—while keeping your reducers pure and synchronous.Purpose of EffectsIsolate Impure Logic: Reducers must remain pure (no side effects, just compute new state from current state and actions). Effects handle everything impure or asynchronous.
React to Actions: They listen to the stream of dispatched actions and trigger side effects, often resulting in new actions being dispatched.
Integration with RxJS: Effects are powered by RxJS Observables, allowing the use of operators for mapping, switching, error handling, and more, making them ideal for complex async workflows.

How Effects WorkSetup: Install @ngrx/effects and add EffectsModule.forRoot([]) or EffectsModule.forFeature([]) to your Angular module.
Creation: Use createEffect() to define an effect as an Observable that subscribes to the Actions stream (all dispatched actions).
Execution Flow:An action is dispatched (e.g., from a component).
Reducers run first, updating state synchronously.
Effects then process the action if it matches (via ofType operator).
Perform side effect (e.g., HTTP request).
Dispatch new actions (e.g., success/failure) or none if { dispatch: false }.

RxJS Operators Commonly Used:ofType: Filters actions by type.
mergeMap, switchMap, concatMap, exhaustMap: Handle async operations and concurrency (e.g., switchMap cancels previous requests).
map: Transform data.
catchError: Handle errors, often dispatching failure actions.
withLatestFrom or concatLatestFrom: Combine with state selectors.

Effects run after reducers for the initial action, ensuring they can access updated state if needed.Basic ExampleHere's a modern (post-v8) example for loading data from an API:typescript

import { Injectable } from '@angular/core';
import { Actions, createEffect, ofType } from '@ngrx/effects';
import { of } from 'rxjs';
import { catchError, map, mergeMap } from 'rxjs/operators';
import { UserService } from './user.service';
import * as UserActions from './user.actions';

@Injectable()
export class UserEffects {
  loadUsers$ = createEffect(() =>
    this.actions$.pipe(
      ofType(UserActions.loadUsers),
      mergeMap(() =>
        this.userService.getUsers().pipe(
          map(users => UserActions.loadUsersSuccess({ users })),
          catchError(error => of(UserActions.loadUsersFailure({ error })))
        )
      )
    )
  );

  constructor(private actions$: Actions, private userService: UserService) {}
}

This effect listens for loadUsers, calls the service, and dispatches success or failure actions.

For non-dispatching effects (e.g., navigation):typescript

navigateOnSuccess$ = createEffect(() =>
  this.actions$.pipe(
    ofType(UserActions.loadUsersSuccess),
    tap(() => this.router.navigate(['/users']))
  ), { dispatch: false }
);

Here are advanced RxJS operators commonly used in real-world applications (especially in NgRx Effects, complex forms, WebSocket handling, and performance optimization). Each includes a practical scenario, code example, and explanation of why it's powerful.1. switchMap – Cancel Previous Requests (Most Common Advanced Operator)Use case: Search-as-you-type (debounced input), preventing race conditions where slow responses arrive out of order.typescript

searchTerm$: Observable<string> = this.searchInput$.pipe(
  debounceTime(300),
  distinctUntilChanged()
);

results$ = this.searchTerm$.pipe(
  switchMap(term => 
    term ? this.http.get<SearchResults>(`/api/search?q=${term}`) : of([])
  )
);

Why advanced?
switchMap cancels the previous inner Observable (e.g., old HTTP request) when a new emission arrives. Only the latest search result matters.2. mergeMap (aka flatMap) – Parallel ExecutionUse case: Dispatch multiple independent API calls concurrently (e.g., load user profile + posts + friends).typescript

loadEverything$ = this.actions$.pipe(
  ofType(UserActions.loadDashboard),
  mergeMap(({ userId }) => [
    this.userService.getProfile(userId),
    this.postService.getPosts(userId),
    this.friendService.getFriends(userId)
  ].map(obs => obs.pipe(
    catchError(err => of({ type: '[Error] Load Failed', err }))
  ))),
  mergeMap(obs => obs) // Flatten the array of observables
);

Why advanced?
Allows parallel execution with no limit (use with caution to avoid flooding servers).3. concatMap – Sequential Execution (Preserve Order)Use case: Upload files one after another (must finish first before starting next).typescript

fileUpload$ = this.fileQueue$.pipe(
  concatMap(file => this.uploadService.upload(file))
);

Why advanced?
Ensures order: next upload starts only after previous completes (or errors).4. exhaustMap – Ignore New Emissions While BusyUse case: Login button – ignore repeated clicks while login request is in flight.typescript

login$ = this.actions$.pipe(
  ofType(AuthActions.login),
  exhaustMap(action =>
    this.authService.login(action.credentials).pipe(
      map(user => AuthActions.loginSuccess({ user })),
      catchError(error => of(AuthActions.loginFailure({ error })))
    )
  )
);

Why advanced?
Completely ignores new emissions until the current inner observable completes.5. withLatestFrom / concatLatestFrom – Combine with Latest StateUse case: Access current store state inside an NgRx Effect.typescript

addToCart$ = createEffect(() =>
  this.actions$.pipe(
    ofType(CartActions.addItem),
    concatLatestFrom(() => this.store.select(selectUserId)), // NgRx v14+
    mergeMap(([action, userId]) =>
      this.cartService.addItem({ ...action.item, userId })
    )
  )
);

Alternative with vanilla RxJS:typescript

withLatestFrom(this.store.select(selectUserId))

Why advanced?
Combines action stream with latest value from another observable (e.g., store state).6. scan – Accumulate State Like a ReducerUse case: Build local state outside NgRx (e.g., shopping cart in service).typescript

cartItems$ = this.itemAdded$.pipe(
  scan((cart, newItem) => {
    const existing = cart.find(i => i.id === newItem.id);
    if (existing) {
      return cart.map(i => i.id === newItem.id ? { ...i, quantity: i.quantity + 1 } : i);
    }
    return [...cart, { ...newItem, quantity: 1 }];
  }, [] as CartItem[])
);

Why advanced?
Mini-reducer in stream – great for local component state or complex aggregations.7. shareReplay(1) – Hot Observable with ReplayUse case: Share expensive HTTP request across multiple subscribers, cache last result.typescript

userProfile$ = this.http.get<User>('/api/profile').pipe(
  shareReplay(1),     // All subscribers get same result + cache last emission
  catchError(err => {
    this.router.navigate(['/login']);
    return throwError(err);
  })
);

Why advanced?
Turns cold observable (HTTP) into hot + cached. Critical for performance.8. partition – Split Stream into TwoUse case: Separate success and error streams.typescript

const [success$, error$] = partition(
  this.actions$.pipe(ofType(DataActions.loadData)),
  action => !action.error
);

Why advanced?
Cleanly splits one stream into two based on predicate.9. bufferTime / bufferCount – Batch EmissionsUse case: Bulk save changes every 2 seconds.typescript

autoSave$ = this.changes$.pipe(
  bufferTime(2000),
  filter(buffer => buffer.length > 0),
  mergeMap(changes => this.saveChanges(changes))
);

10. retryWhen + delayWhen – Advanced Retry with Backofftypescript

resilientRequest$ = this.http.get('/api/data').pipe(
  retryWhen(errors =>
    errors.pipe(
      scan((acc, error) => {
        if (acc >= 5) throw error; // Max 5 retries
        return acc + 1;
      }, 0),
      delayWhen(retryCount => timer(retryCount * 1000)) // Exponential backoff
    )
  )
);

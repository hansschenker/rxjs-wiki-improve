## Description

After task_01 adds the `missing-concept-page` lint check, this task wires up the auto-fix path and ensures the lint UI handles the new issue type properly.

### Implementation

**1. Add `fixMissingConceptPage()` in `src/lib/lint-fix.ts`:**

The fix for a "missing concept page" issue is to generate a stub wiki page for the concept. Strategy:

- Parse the concept name from the issue message (it follows the pattern: `Concept "X" is mentioned in ... but has no dedicated page`)
- Use `slugify()` to create a slug from the concept name
- Check the slug doesn't already exist (guard against race conditions between lint and fix)
- Call the LLM with a prompt like: "Create a concise wiki page for the concept '{concept}'. Include a brief definition and note that this page was auto-generated from a lint suggestion and may need expansion."
- If no LLM key is available, generate a minimal stub: `# {Concept}\n\n*This page was auto-generated by lint. Expand it with real content by ingesting sources about this topic.*`
- Write the page via `writeWikiPageWithSideEffects()` with `logOp: "ingest"` and `crossRefSource` set to the content (so cross-refs are discovered)

**2. Add the case to `fixLintIssue()` switch:**

```ts
case "missing-concept-page":
  return fixMissingConceptPage(issue);
```

**3. Update `src/app/lint/page.tsx`:**

The lint UI already handles displaying issues and calling the fix endpoint. Verify that:
- The new `"missing-concept-page"` type renders properly (it should since the UI maps over issues generically)
- The fix button works for this type
- Add an appropriate icon/label for the new type if the UI uses type-specific rendering

Check the lint page for any type-specific switch/map that needs updating.

**4. Add tests in `src/lib/__tests__/lint-fix.test.ts`:**

- Test `fixMissingConceptPage` generates a stub page when no LLM key
- Test it calls `callLLM` when a key is available (mock callLLM)
- Test it refuses to overwrite an existing page with the same slug
- Test it returns a proper `FixResult`

Title: Markdown rendering for wiki pages
Files: package.json, src/app/wiki/[slug]/page.tsx, src/components/MarkdownRenderer.tsx
Issue: none

## Description

Wiki pages currently display raw markdown in a `<div class="whitespace-pre-wrap font-mono">`. This means headings, links, lists, bold/italic, and code blocks all render as plain text. The wiki is largely useless to read this way.

### Install `react-markdown` + `remark-gfm`

```sh
pnpm add react-markdown remark-gfm
```

These are lightweight, well-maintained, and work perfectly with React Server Components (react-markdown v9+ supports RSC).

### Create `src/components/MarkdownRenderer.tsx`

Create a reusable component:

```tsx
import ReactMarkdown from "react-markdown";
import remarkGfm from "remark-gfm";

interface MarkdownRendererProps {
  content: string;
}

export function MarkdownRenderer({ content }: MarkdownRendererProps) {
  return (
    <div className="prose prose-neutral dark:prose-invert max-w-none">
      <ReactMarkdown remarkPlugins={[remarkGfm]}>{content}</ReactMarkdown>
    </div>
  );
}
```

The `prose` class comes from `@tailwindcss/typography`. We need to add that too:

```sh
pnpm add @tailwindcss/typography
```

Then add the plugin to the Tailwind config. Check what the current tailwind.config.ts looks like — if it uses Tailwind v4 (CSS-based config), add the typography plugin via CSS import in `globals.css`:

```css
@import "tailwindcss";
@plugin "@tailwindcss/typography";
```

If it's a JS config, add to the plugins array.

### Update `src/app/wiki/[slug]/page.tsx`

Replace the raw text display:

```tsx
// OLD:
<div className="whitespace-pre-wrap font-mono text-sm leading-relaxed">
  {page.content}
</div>

// NEW:
<MarkdownRenderer content={page.content} />
```

Import the component from `@/components/MarkdownRenderer`.

### Cross-reference link handling

Wiki pages generated by the LLM may contain markdown links to other wiki pages (e.g., `[Related Concept](related-concept.md)`). These `.md` links should ideally resolve to `/wiki/related-concept`. For now, this can be a follow-up — just getting markdown rendered is the priority. But if time allows, add a custom link component to ReactMarkdown that rewrites `.md` hrefs:

```tsx
components={{
  a: ({ href, children, ...props }) => {
    // Rewrite internal .md links to /wiki/ routes
    if (href && href.endsWith('.md') && !href.startsWith('http')) {
      const slug = href.replace(/\.md$/, '');
      return <a href={`/wiki/${slug}`} {...props}>{children}</a>;
    }
    return <a href={href} {...props}>{children}</a>;
  }
}}
```
